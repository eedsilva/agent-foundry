## Coverage rule

`validateRoadmap` fails when a Personal v1 requirement has an unknown milestone, unknown task, task outside its declared milestone set, milestone outside the transitive v1 path, empty statement or empty evidence. The current contract has twelve normative capability groups and all twelve are structurally covered.

Structural coverage supports a planning-confidence statement; it does not prove implementation quality. Release confidence requires the live Issue Radar evidence named above.

## Explicitly deferred

The following decisions are represented as non-goals rather than silent gaps:

- existing-repository import, Linux host support and browser code editor: post-v1 `v0.4` milestone;
- Windows host support: `personal-v3` milestone;
- hosted multi-tenancy, billing, collaboration and public operations: Hosted Platform v2 track;
- Vercel, Supabase Cloud, cloud VPS provisioning, OAuth, SMTP, automatic DNS mutation, image generation and database auto-rollback: no committed milestone until the product contract changes.
