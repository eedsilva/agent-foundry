# Issue #256: Supabase Harness Conventions and Scaffolds Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give every generated Next.js+Supabase app a dedicated `harness/stacks/supabase.md` convention doc, pinned framework versions in `harness/stacks/nextjs.md`, and a set of static Supabase auth/client scaffold files that get written into a project's workspace at creation time — verifiably template-sourced, not LLM-generated.

**Architecture:** Two independent surfaces feed the same seam. (1) Static markdown content wired through the existing `harness/manifest.json` fragment-selection mechanism (`VersionedHarnessRepository.select`) — no code change, pure content. (2) A new `HarnessRepository.scaffoldFiles(stack)` port method reads static files from `harness/scaffolds/<stack>/`, and a new `WorkspaceManager.applyScaffold(projectId, files)` port method writes them verbatim into the project's real git workspace. `ProjectService.create()` wires the two together right after `writePrd`, before any LLM call ever runs, and records an artifact (`scaffold-manifest`, `createdBy: scaffold:<stack>`) and a `scaffold.applied` event as the run-artifact provenance the issue's acceptance criteria require.

**Tech Stack:** TypeScript, Node.js >=22, vitest, zod, existing `@agent-foundry/{domain,harness,persistence,orchestrator,contracts}` packages.

## Global Constraints

- Node >=22 (repo `package.json` engines). `readdir(dir, { recursive: true })` and top-level `stat` from `node:fs/promises` are available — no new dependency needed.
- `harness/` is NOT excluded from this repo's own `eslint . --max-warnings=0` or `prettier --check .` (root `eslint.config.mjs` / `.prettierignore` do not list `harness/`). - `stack` is a free-form field on `WorkflowDefinition.stack` (`packages/contracts/src/workflow.ts:148`, `PathSegmentSchema`). Today the only production workflow (`workflows/web-app-v1.yaml:5`) sets `stack: nextjs`. Scaffold and harness-fragment content is keyed on this exact string `"nextjs"`.

```markdown
# Stack: Supabase backend conventions

Applies whenever the golden stack's backend is the project's isolated local Supabase Docker stack (ADR 0007). Every generated project uses exactly one Supabase instance; never point a generated app at Supabase Cloud or another project's stack.

## Migrations

- Forward-only. Never edit an applied migration; add a new one.
- One file per change under `supabase/migrations/<timestamp>_<name>.sql`, timestamp `YYYYMMDDHHMMSS`.
- Postgres DDL is transactional by default; only wrap a migration in explicit `begin;`/`commit;` when the statement set specifically requires it.
- Never generate a `down` migration. Rollback is a new forward migration that reverses the change.

## Row Level Security

- Enable RLS on every table holding user data: `alter table <table> enable row level security;` in the same migration that creates the table.
- Default-deny: no table is queryable until a policy exists. Write explicit `select`/`insert`/`update`/`delete` policies scoped to `auth.uid()`.
- Name policies `<table>_<operation>_<scope>` (e.g. `tasks_select_owner`) so intent is legible in `supabase db diff`.
- Never grant `anon` write access to a user-data table.

## Auth

Every app ships with Supabase auth wired end to end: sign-in flow, protected routes, and session handling.

- Email/password only. No OAuth, magic links, or SMTP; password reset is an administrator operation (per ADR 0007 / `docs/PRODUCT_CONTRACT.md`).
- Use `@supabase/ssr` for session handling: a browser client (`createBrowserClient`) for client components and a server client (`createServerClient`) that reads/writes cookies for server components, route handlers, and middleware.
- Protect routes in `middleware.ts` by refreshing the session and redirecting unauthenticated requests away from authenticated segments; never gate authorization in client components alone.
- Store the signed-in user's id as the row owner column (`user_id uuid references auth.users(id) default auth.uid()`) so RLS policies can reference it directly.

## Storage

- Create a bucket per logical asset type, not one shared bucket.
- Default every bucket to private; add a storage policy mirroring the table-level RLS pattern (`bucket_id = '<name>' and owner = auth.uid()`).
- Never mark a bucket public unless the PRD explicitly calls for public asset delivery.

## Environment and secrets

- Local Supabase URL and keys live in `.env.local`, git-ignored; ship `.env.example` with variable names only, never values.
- The service-role key never leaves server-only code paths (route handlers, server actions); it must not be imported into any file reachable from a client bundle.
- Reference `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` for the browser client; the service-role key uses a non-`NEXT_PUBLIC_` name.
```

```typescript
import { createServerClient } from '@supabase/ssr';
import { NextResponse, type NextRequest } from 'next/server';

const PUBLIC_PATHS = ['/sign-in', '/sign-up'];

export async function middleware(request: NextRequest) {
  let response = NextResponse.next({ request });

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() {
          return request.cookies.getAll();
        },
        setAll(cookiesToSet) {
          for (const { name, value } of cookiesToSet) {
            request.cookies.set(name, value);
          }
          response = NextResponse.next({ request });
          for (const { name, value, options } of cookiesToSet) {
            response.cookies.set(name, value, options);
          }
        },
      },
    },
  );

  const {
    data: { user },
  } = await supabase.auth.getUser();

  const isPublicPath = PUBLIC_PATHS.some((path) => request.nextUrl.pathname.startsWith(path));
  if (!user && !isPublicPath) {
    return NextResponse.redirect(new URL('/sign-in', request.url));
  }

  return response;
}

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)'],
};
```

Add this method to the `VersionedHarnessRepository` class, after `select(...)` and before `safeResolve`:

```typescript
  async scaffoldFiles(stack: string): Promise<Array<{ path: string; content: string }>> {
    const scaffoldRoot = this.safeResolve(join('scaffolds', stack));
    let entries: string[];
    try {
      entries = await readdir(scaffoldRoot, { recursive: true });
    } catch (error) {
      if ((error as NodeJS.ErrnoException).code === 'ENOENT') return [];
      throw error;
    }
    const files: Array<{ path: string; content: string }> = [];
    for (const entry of entries.sort()) {
      const fullPath = resolve(scaffoldRoot, entry);
      if ((await stat(fullPath)).isFile()) {
        files.push({
          path: entry.split(sep).join('/'),
          content: await readFile(fullPath, 'utf8'),
        });
      }
    }
    return files;
  }
```

  async applyScaffold(
    projectId: string,
    files: Array<{ path: string; content: string }>,
  ): Promise<{ written: string[] }> {
    await this.ensure(projectId);
    const workspace = this.workspacePath(projectId);
    const written: string[] = [];
    for (const file of files) {
      if (isAbsolute(file.path)) throw new Error(`Unsafe scaffold path: ${file.path}`);
      const destination = join(workspace, ...file.path.split('/').map(safeSegment));
      await ensureDir(dirname(destination));
      await atomicWriteText(destination, file.content);
      written.push(file.path);
    }
    return { written };
  }
