# Design standard

No fixed house aesthetic: derive the visual direction from the PRD and its audience, then execute it consistently.

- Define one spacing scale, type scale, and color palette and reuse them across the app.
- Support light and dark mode from the start.
- Every user-facing workflow has designed loading, error, and empty states. No dead ends or raw exception text.
- Layouts are responsive; primary flows work at mobile widths.
- Pragmatic accessibility: semantic HTML, keyboard navigability, visible focus, readable contrast. A formal WCAG audit is not required unless the PRD asks.
- Finished over flashy: aligned edges, consistent radii and shadows, real copy instead of placeholder text.
